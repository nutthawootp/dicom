# DICOM Batch De-identification Tool

เครื่องมืออัตโนมัติสำหรับทำความสะอาดและลบข้อมูลระบุตัวตน (De-identification / Anonymization) ออกจากไฟล์ภาพทางการแพทย์ (DICOM) ออกแบบมาเพื่อสนับสนุนงานวิจัยทางคลินิก (Clinical Trials) ช่วยลดระยะเวลาในการเตรียมเอกสาร ลดความเสี่ยงจาก Human Error และรักษามาตรฐานความปลอดภัยของข้อมูลตามหลัก GCP (Good Clinical Practice)

## ✨ คุณสมบัติเด่น (Key Features)
* **Interactive GUI Workflow:** ทำงานผ่านหน้าต่างโต้ตอบที่ใช้งานง่าย บังคับให้ตรวจสอบความถูกต้องของข้อมูลก่อนเริ่มทำงานเสมอ
* **Pre-scan Data Validation:** ดึงข้อมูล Metadata จากไฟล์ต้นฉบับมาแสดงผลให้ตรวจสอบ (Cross-check) ก่อนทำการกรอกข้อมูลใหม่
* **Recursive Folder Processing:** สแกนหาไฟล์ DICOM ในทุกโฟลเดอร์ย่อย (Subdirectories) โดยอัตโนมัติ และสร้างโครงสร้างโฟลเดอร์ปลายทาง (Output Directory Hierarchy) ให้เหมือนต้นฉบับทุกประการ
* **Advanced PHI Removal:** ลบข้อมูลความลับผู้ป่วย รวมถึงเส้นวาดระยะและกล่องข้อความ (Measurements & Annotations) ที่ฝังอยู่ใน Metadata
* **Non-destructive Process:** ไม่มีการเขียนทับ (Overwrite) ไฟล์ต้นฉบับเด็ดขาด ไฟล์ที่ผ่านการประมวลผลจะถูกบันทึกแยกต่างหากพร้อมสร้างโฟลเดอร์ใหม่ให้อัตโนมัติ
* **Automated Summary Report:** สรุปยอดรวมจำนวนไฟล์ที่ทำงานสำเร็จ, ไฟล์เอกสารที่ถูกข้าม, และไฟล์ที่พบข้อผิดพลาดเมื่อจบกระบวนการ

## 🛠 ข้อมูลทางเทคนิคและสิ่งที่ต้องติดตั้ง (Prerequisites)
เครื่องมือนี้พัฒนาด้วยภาษา Python โดยใช้ระบบจัดการ Dependency ที่เน้นความรวดเร็วและเสถียรอย่าง `uv`
1. **Python:** เวอร์ชัน 3.8 ขึ้นไป
2. **uv:** สำหรับจัดการ Environment และ Dependency
3. **Libraries:**
   * `pydicom`: สำหรับอ่านและแก้ไข DICOM Tags
   * `tkinter`: สำหรับสร้าง GUI (ติดตั้งมาพร้อมกับ Python มาตรฐาน)

**การติดตั้งเบื้องต้น (Setup):**
```bash
# สร้างโปรเจกต์และ Environment ใหม่
uv init dicom-deidentifier
cd dicom-deidentifier

# ติดตั้ง Dependency
uv add pydicom
```

## 🚀 วิธีการใช้งาน (Usage Guide)
1. รันสคริปต์ด้วยคำสั่ง `uv run main.py`
2. **ขั้นตอนที่ 1:** โปรแกรมจะเปิดหน้าต่างให้เลือกโฟลเดอร์ที่เก็บไฟล์ DICOM ต้นฉบับ
3. **ขั้นตอนที่ 2:** โปรแกรมจะแสดงหน้าต่างระบุพารามิเตอร์ 
   * ตรวจสอบ "ข้อมูลต้นฉบับ" ในกรอบด้านบนเพื่อให้แน่ใจว่าเลือกเคสถูกต้อง
   * กรอก `Subject Number` และ `Protocol Number` ในช่องที่กำหนด
4. **ขั้นตอนที่ 3:** หน้าต่างสรุปข้อมูลจะแสดงขึ้นมา ตรวจสอบความถูกต้องอีกครั้งแล้วกด **"Run (เริ่มทำงาน)"**
5. รอจนกว่ากระบวนการจะเสร็จสิ้น โปรแกรมจะแสดงกล่องข้อความสรุปผลการทำงาน

## 📋 ข้อกำหนดการแก้ไขข้อมูล (De-identification Specifications)
สคริปต์นี้ถูกตั้งค่าให้แก้ไขและลบ DICOM Tags ตามมาตรฐานดังต่อไปนี้:

**1. ข้อมูลที่ถูกแทนที่ (Replaced):**
* `(0010,0010) Patient Name` ➔ แทนที่ด้วย **Subject Number**
* `(0010,0020) Patient ID` ➔ แทนที่ด้วย **Protocol Number**
* `(0008,0080) Institution Name` ➔ แทนที่ด้วย **Protocol Number**
* `(0008,0090) Referring Physician Name` ➔ แทนที่ด้วย **Protocol Number**

**2. ข้อมูลที่ถูกปรับเปลี่ยน (Modified):**
* `(0010,0030) Patient Birth Date` ➔ คงเหลือเฉพาะ "ปีเกิด" ส่วนวันที่และเดือนถูกเปลี่ยนเป็น `0101` (เช่น `19850101`) เพื่อประโยชน์ในการคำนวณอายุขณะเข้าร่วมโครงการ

**3. ข้อมูลที่ถูกล้างค่า / ลบทิ้ง (Stripped / Deleted):**
* `(0008,0081) Institution Address`
* `(0008,1050) Performing Physician Name`
* `(0010,0040) Patient Sex`
* `(0010,1010) Patient Age`
* `(0070,0001) Graphic Annotation Sequence` (ข้อมูลการวาดเส้น/กล่องข้อความบนภาพ)
* `Group (5000-50FF)` (ข้อมูล Curve Data)
* `Group (6000-60FF)` (ข้อมูล Overlay Data)

## 📁 โครงสร้างผลลัพธ์ (Output Structure)
โปรแกรมจะสร้างโฟลเดอร์ปลายทางให้อัตโนมัติในระดับชั้นเดียวกันกับโฟลเดอร์ต้นทาง โดยใช้รูปแบบชื่อดังนี้:
`[ชื่อโฟลเดอร์ต้นทาง]_DeID_[Subject_Number]`

ไฟล์ที่ถูก De-identified แล้วจะถูกเปลี่ยนชื่อโดยนำ `Subject_Number` มานำหน้าชื่อไฟล์เดิมเสมอ เพื่อป้องกันความสับสน (เช่น `SUBJ-001_image001.dcm`)
